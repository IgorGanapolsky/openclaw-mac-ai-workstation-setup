# Fulfillment Checklist

## Before Purchase

- Confirm buyer has a Mac and wants a local AI-agent workstation.
- Confirm they accept that macOS permissions require human approval.
- Confirm no guaranteed revenue, job, trading, or account outcomes.

## Intake

- Collect OS version, machine type, shell, package manager.
- Collect current tools and exact commands that fail.
- Ask for sanitized logs only. Do not request secrets in chat.
- If keys are needed, have buyer enter them locally while screen-sharing or use their secret manager.

## Setup Steps

- Back up config files before editing.
- Verify OpenClaw CLI version and gateway health.
- Verify model status and auth profile names without exposing values.
- Remove or isolate stale provider profiles.
- Configure main agent for coding/business work.
- Configure Computer Use path:
  - macOS Screen Recording granted.
  - macOS Accessibility granted.
  - Codex Computer Use or Peekaboo ready.
  - Real app-inspection smoke test completed.
- Configure per-directory TUI wrapper if OpenClaw TUI reuses sessions.
- Disable noisy memory/search paths if invalid keys block normal work.
- Verify one coding/business text smoke.
- Verify one desktop automation smoke.

## Evidence To Deliver

- Gateway health result.
- Model route summary.
- Auth provider summary with secrets redacted.
- Mac permissions readback.
- Computer Use smoke result.
- Changed file list.
- Backup file list.
- Short operating runbook.
